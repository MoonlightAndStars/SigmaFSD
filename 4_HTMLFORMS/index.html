HTML FORMS

Forms in HTML are a method of getting user data through user input and use the obtained data to process the desired operation by submitting the data from the client side to the server side. Forms in HTML are created by "<form></form>" tag

Form Input Types

  1. text
  2. radio
  3. checkbox
  4. textarea
  5. submit
  6. password
  7. email
  8. tel
  9. number
  10. color

Radio Tag

  1. Radio inputs require a <label> to make them more accessible. If they are not linked to a label, clicking on the text will not select the radio button.
  2. input type radio creates radio boxes
  3. these radio boxes/buttons are clickable only one at a time and we cannot multi-select radio buttons
  4. here in the below example there are two radio buttons one for male and another for female and both are linked to their respective labels through IDs.
  5. The name attribute identifies the input field when the form is submitted. The server uses this name to retrieve the value entered by the user. All input fields with the same name are treated as a group, allowing only one selection at a time.
  6. here in this example the name is set to gender for both the input fields.
  7. the value field defines the actual data or value that will be sent to the server if the input is selected or filled.
  8. for checkboxes and input buttons the value is sent only if the input is checked.

Input Type "password" Example

In an input type="password", the entered password is masked on the screen. To secure password data, forms should use the POST method instead of GET to prevent exposure in the URL. Additionally, hashing mechanisms in JavaScript or backend security measures should be used before storing passwords.

We can use attribute like minlength and maxlength in input to tell the browser the length of the value expected in the field.
